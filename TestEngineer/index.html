Test Engineer Assessment

  1. Home
  2. Test Engineer

In this assessment you will write a few tests against a simple, web-based sample application. We will then review and discuss your code in a technical interview.

We know your time is valuable, so we're only asking for 2 to 3 hours on this. If you can't finish the whole thing, that's OK! Do what you can. (You're welcome to spend more time if you want, but it's not required.)

If you already have samples of your UI tests that you can share, those are acceptable too!

Want to use your own dev environment?

You can do this assessment on your own computer. Just clone the Heuristics.TestEngineerEval GitHub repo.

Starting the sample app

We have pre-installed the eShopOnWeb sample app on this virtual machine.

It should be automatically started and available at https://localhost:5001. There are bookmarks to this app in the Chrome browser.

If the web app isn't running, check the d:\TechAssessment\__START_ESHOP.bat file for the sample instructions to get it running.

Writing the Tests

We have created an empty C# + Nunit + Selenium project in d:\TechAccessment\TestEngineer\ for you to work in. You can open it with Visual Studio.

View the instructions on GitHub for the specific tests to write.

To run your tests, use the Test -> Run All Tests menu in Visual Studio.

Frequently Asked Questions

If you're wondering about something, chances are someone else has wondered about it too!

  • I'm not familiar with .NET; what should I do?

    That's awesome! This is a tremendous opportunity for you to demonstrate how you'd approach a problem in an unfamiliar environment, which is exactly what we want to know! We're more interested in your ability to learn than what you know right this second.

    Using Selenium in .NET is similar to using Selenium in Java, and we've already set up the project for you. Give it your best shot, and don't be afraid to look up some introductory C# tutorials if you're completely lost.

  • Can I just send in an existing code sample instead of doing this assessment?

    If you have code you're proud of and can legally share with us, we're happy to start there! Reach out to your hiring manager for details.

  • Should I use tabs or spaces in my sample code?

    Well, of course there is only one correct answer to this question, but you're welcome to use either for your sample code. Once you join the team, however, we'll make sure to indoctrinate you into the One True Way of thinking about whitespace...
