About This Workspace

We do things a little differently at Heuristics

We don't believe that whiteboard coding, timed multiple-choice tests, or asking tricky questions under pressure are effective ways to hire great people.

Instead, we want you to show us how you think and what you can do while working at your own pace, with access to reference materials, and without us looking over your shoulder; just like you'd be working in real life!

This workspace gives you a fully-set-up development environment in which to do that. You don't need to install or run anything on your local computer. This environment includes:

  • Visual Studio 2022 - Community Edition
  • SQL Server - Developer Edition
  • SQL Server Management Studio
  • IIS
  • All of our sample apps, ready to go

You are authorized to use this workspace only for the purposes of taking the Heuristic Solutions technical assessments.

Don't want to use this workspace? That's cool; all of our assessments are on GitHub too. See the links within each assessment's instructions.

Our tech stack

We've been building our app for a long time, so we use a lot of different stuff.

  • ASP.NET MVC / .NET Core
  • SQL Server / NHibernate / Dapper
  • VueJS / jQuery
  • nUnit / Jest / Selenium
  • AWS / SQS / Lambdas / RabbitMQ
  • Run everything locally; no shared "development database" nonsense!

We believe in automation, clean code, and doing things right the first time. Click the "learn more" button to see more details about our stack, process, and technical values.

Frequently Asked Questions

If you're wondering about something, chances are someone else has wondered about it too!

  • Do I need to do all of these assessments?

    Not unless you're interviewing for multiple jobs with us :) Just do the one the hiring manager directed you to, which should match the role you're applying for.

  • How long is this supposed to take?

    We know you have a life outside of interviewing, so we're only asking for 2-3 hours of your time. Do what you can within that window; it's OK if you don't finish everything.

    That being said, feel free to go the extra mile if you really want to impress us!

  • Can I just send in an existing code sample instead of doing this assessment?

    If you have code you're proud of and can legally share with us, we're happy to start there! Reach out to your hiring manager for details.

  • Should I use tabs or spaces in my sample code?

    Well, of course there is only one correct answer to this question, but you're welcome to use either for your sample code. Once you join the team, however, we'll make sure to indoctrinate you into the One True Way of thinking about whitespace...
